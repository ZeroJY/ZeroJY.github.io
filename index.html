概念

  • Ajax是Asynchronous Javascript和XML的简写 。即异步JavaScript和XML
  • 用途：动态刷新局部数据，无需卸载整个页面，从而带来更好的用户体验
  • Ajax核心：XMLHttpRequest对象(简称XHR)
    • 我们使用ajax，主要有以下两个特性：
  1. 向服务器端发送请求，而不用重新加载页面。

  2. 从服务器端接收数据并处理。

    如何创建ajax

    1. 创建请求对象

    • 需要通过XMLHttpRequest实现使用JavaScript向服务器端发送一个HTTP请求。
    • 但是有兼容性问题，在IE中我们引入一个名为XMLHTTP的ActiveXObject实现与XMLHttpRequest相同的功能

    2. 格式化数据，如果每次都不想从缓存读取数据，给url添加一个随机数的参数

    3. 向服务器发送请求

    • 如需将请求发送到服务器，我们使用 XMLHttpRequest 对象的 open() 和 send() 方法：

    各参数详解：

    • method ：请求的类型；GET 或 POST
    • url ：文件在服务器上的位置
    • async ：true（异步）或 false（同步）

      其中get和post的区别：

    • get 提交没有请求体，
      get 提交发送到服务器端的请求的数据都在地址栏当中
      不安全,因为get 提交发送到服务器的数据都在地址栏当中
      所以请求发送的数据的大小有限制.
    • post 有请求体，请求体的数据就是客户端发送到服务端的内容的参数数据.
      post 提交的数据都在请求体当中，相对来说安全
      post 提交对发送到服务器端的数据大小没有限制
      post 提交必须有一个请求头
      Content-Type:application/x-www-form-urlencoded
      以后假设表单提交，如果说要发送中文，最好我们都使用post

      异步 - True 或 False？

      AJAX 指的是异步 JavaScript 和 XML（Asynchronous JavaScript and XML）。
      XMLHttpRequest 对象如果要用于 AJAX 的话，其 open() 方法的 async参数必须设置为 true：

    对于 web 开发人员来说，发送异步请求是一个巨大的进步。很多在服务器执行的任务都相当费时。AJAX 出现之前，这可能会引起应用程序挂起或停止。
    通过 AJAX，JavaScript 无需等待服务器的响应，而是：

    • 在等待服务器响应时执行其他脚本

    • 当响应就绪后对响应进行处理

      Async = false

      JavaScript会等到服务器响应就绪才继续执行。如果服务器繁忙或缓慢，应用程序会挂起或停止。

      onreadystatechange 事件

      当请求被发送到服务器时，我们需要执行一些基于响应的任务。
      每当 readyState 改变时，就会触发 onreadystatechange 事件。
      readyState 属性存有 XMLHttpRequest 的状态信息。

      • onreadystatechange:存储函数（或函数名），每当readyState属性改变时，就会调用该函数。
      • readyState:存有 XMLHttpRequest 的状态。从 0 到 4 发生变化。
      • 0: 请求未初始化
      • 1: 服务器连接已建立
      • 2: 请求已接收
      • 3: 请求处理中
      • 4: 请求已完成，且响应已就绪
      • status: 200: “OK” , 404: 未找到页面
        在 onreadystatechange 事件中，我们规定当服务器响应已做好被处理的准备时所执行的任务。
        当 readyState 等于 4 且状态为 200 时，表示响应已就绪：

      • callback 函数是一种以参数形式传递给另一个函数的函数。
        如果您的网站上存在多个 AJAX 任务，那么您应该为创建 XMLHttpRequest 对象编写一个 标准 的函数，并为每个 AJAX 任务调用该函数。
      • 该函数调用应该包含 URL 以及发生 onreadystatechange 事件时执行的任务（每次调用可能不尽相同）

面向对象的特征、继承、原型链.

发表于 2016-10-06

面向对象的三大特性

  1. 封装 : 将复杂的实现过程包装，隐藏起来，然后提供给用户使用的接口即可。
    优点：
    1. 实现了代码的复用;
    2. 保持函数或者对象功能的单一性，便于日后维护。
  2. 继承 :
    • 在JS中，继承指的是对象之间的关系。
    • 在面向对象的语言中，继承指的是类之间的关系。
    • 实质：构造函数和类都是实例化对象的模板。
    • 概念：就是指一个对象有权去访问另一个对象的成员。
      在JS中，只要让一个对象去访问另一个对象的成员，就必须建立继承关系。
任何对象都继承自它自己的原型对象

  3. 多态
    概念：在父类中的同一方法在各个子类中的实现行为不同。

    继承的实现方式有两种实现的方式：

    1. 基于原型

    • 扩展原型：在原有的原型上进行相应的扩展，实现继承。

    • 置换原型：置换原型:将要被继承的对象，直接替换掉原有的原型，实现继承

  1. 拷贝继承：将指定对象上的所有成员拷贝一份，并添加到当前对象上。这种继承方式成为拷贝继承。

Canvas基础

发表于 2016-10-04

1. Canvas简介：

  • 概念：H5提供的一个新的标签元素。中文：画布
  • Canvas是指可以在其上面通过脚本语言（通常为JS）绘制图形、图像以及制作动画的标签。
  • 本身不具备绘制能力，只是负责展示。
  • 在浏览器不支持canvas时，会将其当做dav标签来解析。
  • 在IE中，会将其当做文本节点来解析

    2. 基本使用

  • 创建一个画布

    • 在创建一个画布是，如果没有指定宽和高，默认为300*150(px)
    • 用行内样式来指定canvas的宽和高；如果在非行内样式指定canvas的宽和高，会在默认值的基础上进行缩放。而在缩放时，并没有增加canvas内部的像素点的个数。

  • 获取绘制工具

    • getContext方法：参数为 ‘2d’ or ‘WebGL’
      2d: 获取绘制平面图形的工具
      WebGL: 获取绘制立体图形的工具
    • 在课程内，指学习绘制2D图形
    • 绘制工具是有归属的。只有画布自己的绘制工具 可以在本身上绘制图形、图像。
  • 绘制工具的API方法
    • 移动笔触 moveTo(x,y) x 确定点的横坐标 ； y 确定点的纵坐标 ；
    • 画线 lineTo(x,y) 从当前笔触所在点 连线 到 指定点。
  • 绘制直线步骤
    • 移动笔触 moveTo
    • 画线 lineTo
    • 描边 stroke 或 填充 fill
  • 坐标系
    • 原点 在cnavas的左上角；
    • X轴： 在原点的水平方向；
    • Y轴： 在原点的垂直方向；

      3. 路径的概念

  • ctx.beginPath();开启路径
  • ctx.closePath();闭合路径
    • 将图形的起始点与终止点进行连线，形成一个封闭的的图形
    • 将自动处理封闭图形的锯齿
  • 只有在需要开启新路径时调用beginPath;
    在需要将图形闭合，形成封闭的图形，那么就可以调用closePath方法
  • 这两个图形并不一定成对出现
  • 除了stroke或fill开头的方法之外，都会绘制路径。此时需要手动调用该方法。

    4. 线性相关的属性

  • lineWidth：设置线宽
  • strokeStyle：指定线条的颜色，值可以为 16进制、颜色字符串、rgb、rgba、渐变色
  • fillStyle：设置填充颜色。
  • lineCap：设置线段两端的样式
    • ‘butt’: 默认线头的两端没有任何样式
    • ‘round’: 设置线段的两端为突出的圆角图形。
    • ‘square’:设置线段两端为突出的方形图形。
  • lineJoin：设置线段相交点的样式
    • ‘round’: 圆角样式
    • ‘bevel’:平角样式
    • ‘miter’:尖角样式

      5. 非零环绕原则

  • 目的：确定某一区域是否需要填充
  • 描述：定义计数默认为0，从当前区域选取任意点，然后从该点到整个图形外部绘制一条射线。观察和这条射线相交的所有线段。
    • 如果该相交的线段是顺时针穿过这条射线，那么计数+1； 如果是逆时针穿过那么计数 - 1；
    • 如果计数为0，那么就不填充该区域，
    • 如果计数非0，那么就填充该区域。

函数的介绍

发表于 2016-10-04

1. Function的介绍

  1. 创建函数的方式
    • 声明式
    • 表达式
    • 构造函数：Function(arg1,arg2,…argN,body)所有参数的类型均为字符串;arg1,arg2,…argN为可选的参数列表，表示为生成函数的形参列表。
    • body为必选参数，表示为生成函数的函数体部分。
  2. 由于函数也可以通过Function构造函数来创建，那么函数也是对象；具有 __proto__ 属性。

    2. 函数的原型链

    • 所有函数都是Function实例。
    • 所有的函数的构造函数为Function>
    • 所有的函数的原型对象为Function.prototype.
    • 将函数成为函数对象
    • 其他对象成为普通对象

      3. arguments对象

    • arguments对象属性
    • length 实参个数
    • callee
    • 返回正在执行的函数;应用在匿名函数递归调用;
    • 函数相关属性
    • caller 返回调用当前函数的函数。
    • length 记录形参的个数。
    • name 函数的名字

      4. 模拟函数重载

  • 在其他的后台语言中，允许函数重载（当有重名函数时，可根据传递参数的个数不同将函数定义为不同的函数
  • 然而在JS中，不允许同名函数，因此我们可以模拟函数重载。

    5. 递归调用

    • 概念：函数调用自身
    • 经典递归案例
    • 等差数列第n项值
    • fibonacci数列第n项值

      6. 词法作用域

    • 规定变量的作用域是由变量声明的位置决定，而不是由使用变量的位置决定。
    • 在词法作用域中，只有函数可以分割作用域。在函数内部是一个完整作用域。与外界隔离。
    • 也就是，在函数内部可以访问函数内部的数据；但是函数外部无法访问内部的数据。

      7. 变量名（函数名）提升

  • 在js预解析阶段执行的操作。
    js是边解析边执行的。

    8. 在预解析阶段

    首先。分析代码语法，如果有代码语法错误，就直接抛出异常。
    其次，将变量名以及函数声明提升到当前作用域最顶端。

    9. in运算符

    规则：前面的属性名字，要用字符串来表示；如果为变量，那么会将变量的值隐式转换为字符串，然后去判断变量的值是否为制定对象的属性（可以为继承过来的属性）
    注意：后面的参数必须为对象；否则会报错。

函数的扩展以及闭包

发表于 2016-10-04

1. instanceof运算规则：

  • 如果运算符后面的函数的prototype属性所引用的对象。
  • 出现在运算符前面的对象的原型链上的话，就返回true；
  • 否则就返回false。

    2. 绘制作用域链的规则

  • 将整个script标签的全局作用域，定义为0级作用域链。将全局作用域上的数据（变量、函数、对象）绘制在该链上。
  • 由于在词法作用域中，只有函数可以分割作用域。因此，只要遇到函数，就要引申出新的作用域链，级别为当前作用域链的级别+1；
  • 重复第二步，直到没有遇到函数为止

    3. 变量的搜索原则：当访问一个变量时。

  • 首先在当前变量所处作用域链上查找，如果找到就直接使用，并停止查找；
  • 如果没有找到，就向上一级链（T-1)上去查找，如果找到就直接使用，并停止查找；
  • 如果没有找到，就继续向上一级链查找。直到0级链，如果找到就直接使用，并停止查找；
  • 如果没有找到就报错。
    注意
  1. 如果访问的变量不存在，会搜索整个作用域链（不仅性能低，而且抛出异常。）
    在实际开发，不推崇所有数据都写在全局上。尽量使用局部变量。推荐使用沙箱模式
    如果在开发中，所有js代码都写在全局上，会造成全局变量的污染。
  2. 同级别的链上的变量互不干扰。

    4. 函数执行的环境

  • 在函数执行的时候，会创建一个执行的环境，包括activeObject(活动对象)以及作用域链
  • activeObject：存储的是所有在函数内部定义的变量，以及函数的形参；
  • 会将变量名字以及形参名字作为该对象的属性来存储。

    5. 闭包

  • 概念：指一个函数有权去访问另一个函数内部数据
  • 缺点：会造成函数内部数据常驻内存，从而引发内存泄漏的问题。
  • 目的：解决在函数外部无法访问函数内部的数据的这个问题。可以说是一种技术手段。
  • 为了防止闭包导致内存泄漏，在使用完闭包之后，将其赋值为null;
  • 闭包怎么缓存数据？因为在函数内部有方法（函数）对其有引用，并且又将其返回到外部作用域上的一个变量接收。

    6. 垃圾回收机制 ：GC(Garbage Collection)

  1. 引用计数
    • 如果一个变量被另一个变量引用，那么该变量的应用计数+1；
    • 当这个变量不在引用这个变量时，那么该变量的应用计数-1；
    • GC会在一定时间间隔去查看每一个变量的计数，如果为0，就将其占用的内存回收。
    • 缺点：循环引用：会造成无法回收变量的额内存，已被摒弃
  2. 计数清除法
    • 从当前文档根部（window对象）找一条路径，如果能到达该变量，那么说明此变量正在被使用，此变量就不应该被回收掉。反之，应该回收被占用的内存。
    • 如果某个变量进入执行环境，那么给其标记为“进入环境”；
    • 如果上述执行环境执行完成，被销毁，那么该环境内的所有变量都被标记为“已出环境”
    • 如果上述变量被标记为已出环境，就会被回收掉其占用的内存空间。

函数四种调用模式以及上下文模式

发表于 2016-10-04

函数四种调用模式

  • 目的：确定this的指向。

    1. 普通函数执行：this -> window

    2. 方法调用:this -> 该方法的调用者

    3. 构造函数模式：this -> 当前要创建出来的实例（对象）

    4. call/apply模式（上下文模式）只有函数和对象才有call和apply方法 动态指定某方法或函数执行时，内部的this指向。

    • this -> 用户动态指定的
      • 指定call or apply 方法的第一个参数
    • 区别：
      • .call(thisObj,arg1,arg2,…,argN):thisObj为this的指向。之后为形参列表，就是FN在执行时的实参列表
      • .apply(thisObj,[实参]):thisObj 为this的指向，数组参数为fn在执行时的实参
    • 注意：在非严格模式下，如果thisObj赋值为null或者不传实参，此时this->window对象，就相当于普通函数执行模式

      上下文模式的应用

  • 1 数组的合并
  • 2 借调方法（函数）
    • 借用函数
    • 借用原生方法

绘制图像以及坐标轴的平移

发表于 2016-10-04

1. 绘制图像drawImage

  • 普通的渲染图像
    • ctx.drawImage(imgObj, x, y)
    • 在canvas的（x,y)位置上，渲染指定的imgObj图像
  • 缩放绘制图像
    • ctx.drawImages(imgObj, x, y, width, height)
    • 在canvas的(x, y)位置上，根据指定的宽和高，来渲染imgObj图像
  • 切片绘制图像
    • ctx.drawImage(imgObj, sx, sy, swidth, sheight, x, y, width, height)
    • sx, sy, swidth, sheight : 在源图片上的sx，sy位置切片一个swidth宽、sheight高的图像。
    • x, y, width, height ：将上一步切出来的图像绘制在canvas的x,y位置上，并且指定渲染的宽和高
  • 尽量保持原图像的比例进行渲染图像。防止图像在渲染的时候发生变形。

    2. 坐标系变换

  • ctx.translate(x, y) 平移坐标系
    • 将坐标系的原点 平移到 指定的(x, y)这点。
  • ctx.rotate(radian)
    • 以当前原点旋转坐标系到指定的弧度radian
    • 如果为负数，逆时针旋转
    • 如果为正数，顺时针旋转

      3. 清除坐标系

  • ctx.clearRect(x, y, w, h)
    • 清除由x, y, w, h确定的矩形以内所有的图形。

      4. 状态保存与恢复

  • ctx.save() : 将上一次的状态保存起来
  • ctx.restore(): 恢复最近一次保存的状态
